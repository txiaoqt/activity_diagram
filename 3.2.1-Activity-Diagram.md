# 3.2.1 Activity Diagram

Activity diagrams show how the major LYDO Connect workflows proceed from one action to the next. The updated set groups the system into seven manuscript-level workflows: Account Registration / Login / Policy Agreement, Manage Programs and Events, Program or Event Registration, Participation Tracking / Registration Monitoring, Management of Service Requests / Citizen Desk, Transparency and Financial Records Management, and User, Role, and Audit Log Management.

## Figure 6.1. Activity Diagram of Account Registration / Login / Policy Agreement

```mermaid
flowchart TD
    A([Start]) --> B[User opens LYDO Connect]
    B --> C{Does the user need a new account?}
    C -- Yes --> D[Open sign-up form]
    D --> E[Enter email, password, name, contact number, and barangay]
    E --> F[System validates account and profile details]
    F -- Invalid --> G[Display validation errors]
    G --> D
    F -- Valid --> H[Create authentication account and user profile]
    H --> I[Open sign-in page]
    C -- No --> I
    I --> J[Enter credentials]
    J --> K[System validates credentials and loads role]
    K -- Invalid --> L[Display login error]
    L --> I
    K -- Valid --> M{User role}
    M -- Admin or staff --> N[Redirect to admin portal]
    M -- Youth user or citizen --> O[Load active Terms and Privacy Policy]
    O --> P{Active policy accepted?}
    P -- Yes --> Q[Continue to public user features]
    P -- No --> R[Display policy agreement modal]
    R --> S{User accepts?}
    S -- No --> T[Sign user out]
    S -- Yes --> U[Save policy acceptance record]
    U --> Q
    N --> V([End])
    Q --> V
    T --> V
```

## Figure 6.3. Activity Diagram of Program or Event Registration

```mermaid
flowchart TD
    A([Start]) --> B[Youth user opens a program or event detail page]
    B --> C{Is the user authenticated?}
    C -- No --> D[Prompt user to sign in or create an account]
    D --> B
    C -- Yes --> E{Has the user accepted the active policy?}
    E -- No --> F[Display policy agreement modal]
    F --> E
    E -- Yes --> G{Is registration available?}
    G -- No --> H[Display registration unavailable message]
    G -- Yes --> I[Display registration form with profile defaults when available]
    I --> J[User enters or confirms name, email, contact number, municipality, and barangay]
    J --> K[System validates required fields and PH mobile number format]
    K -- Invalid --> L[Display validation errors]
    L --> I
    K -- Valid --> M{Record type}
    M -- Event --> N[Check event status and capacity]
    M -- Program --> O[Check program publication status]
    N -- Closed or full --> H
    O -- Not published --> H
    N -- Available --> P[Create or update event registration]
    O -- Published --> Q[Create or update program registration]
    Q --> R[Create or reactivate program membership]
    P --> S{External attendance enabled?}
    R --> S
    S -- Yes --> T[Mark registration sync status as pending]
    S -- No --> U[Mark registration sync status as skipped]
    T --> V[Display registration confirmation]
    U --> V
    V --> W([End])
    H --> W
```

## Figure 6.4. Activity Diagram of Participation Tracking / Registration Monitoring

```mermaid
flowchart TD
    A([Start]) --> B{Actor}
    B -- Youth user --> C[Open profile page]
    C --> D[System verifies active session]
    D --> E[Load program memberships and event or program registration records]
    E --> F[Display participation history and current registration statuses]
    F --> G([End])
    B -- Admin or staff --> H[Open Registrations module]
    H --> I[System loads events, programs, local registrations, barangays, source, and sync status]
    I --> J[Apply event or program mode, record filter, and search terms]
    J --> K[Display total rows, unique participants, pending sync count, and filtered table]
    K --> L{Selected admin action}
    L -- Export CSV --> M[Generate CSV from filtered local records]
    L -- Preview external sheet --> N[Normalize Google Sheet URL and fetch published CSV rows]
    L -- Retry failed sync --> O[Set selected row sync status to pending]
    L -- Run sync now --> P[Call protected sync-run-once endpoint]
    P --> Q[Worker processes pending rows and updates sync status]
    M --> R[Admin reviews result]
    N --> R
    O --> R
    Q --> R
    R --> G
```

## Interpretation

- The account workflow combines registration, login, role-based redirection, and active Terms of Service and Privacy Policy acceptance.
- The program and event management workflow covers admin creation, editing, publishing, archiving, scheduling, location, source, and registration-integration settings.
- The registration workflow stores participation in `event_registrations` or `program_registrations`, with sync status set according to external attendance settings.
- Participation tracking is user-facing through the profile page, while registration monitoring is admin-facing through filters, CSV export, external sheet preview, retry, and sync-run actions.
- The citizen desk workflow supports ticket submission, public tracking, account-linked ticket history, admin handling, and audit logging.
- Transparency and financial workflows cover documents, board records, monthly compliance, barangay financials, and youth metrics.
- User, role, and audit log management covers access administration and accountability review.
